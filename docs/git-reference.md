# Git 運用リファレンス

このファイルは要約と参照用のページです。**canonical（一次情報）** は `.claude/skills/git-rules/REFERENCE.md` にあります。エージェントは `REFERENCE.md` を優先して参照してください。

主なポイント（要約）

- ブランチ戦略: GitHub Flow を採用。`main` は常にデプロイ可能な状態に保つ。
- ブランチ命名: `feature/<短い説明>`、`fix/<短い説明>`、`chore/<短い説明>`、`hotfix/<短い説明>` など。
- PR: コミッター以外の承認者を最低 1 人要求。PR 本文に「何を・なぜ・影響範囲・テスト手順」を記載すること。
- コミット: Conventional Commits を推奨。

詳細は canonical を参照してください: `.claude/skills/git-rules/REFERENCE.md`。
