## いつ使うか
- 開発者から「このリポジトリでのブランチルールは？」「このPRはルールに従っているか？」といった問い合わせがあったとき。
- PR 作成時のチェックリストを自動で提示したいとき。

## 手順（人間 / エージェント向け）
1. まず `docs/git-reference.md` を読み、該当するルール（ブランチ命名、PR 承認、CI 要件など）を特定する。
2. ユーザーから PR チェックを依頼された場合は、以下を順に実施する：
   - PR のタイトルが Conventional 書式に合っているか確認
   - PR 本文に「何を」「なぜ」「影響範囲」「テスト手順」が記載されているか確認
   - 少なくとも 1 名の Approver（コミッター以外）がいるか確認
   - CI のステータス（存在する場合）が成功しているか確認
   - 上記のいずれかが不足していたら、PR に対して具体的な修正提案コメントを作成する（例文を提示）
3. 必要なら、簡易的なコマンドや GH API 呼び出しのサンプル（`gh api`, `gh pr view` 等）を提案する。

## 実装のヒント
- SKILL は `docs/git-reference.md` を唯一の権威（canonical）として参照してください。
- `SKILL.md` に書かれたチェックはあくまで助言であり、ブランチ保護（Branch Protection）での強制設定は管理者に委ねてください。

## 例: PR チェックをエージェントに実行させるためのプロンプト
- "この PR (#123) が規約に従っているか、`docs/git-reference.md` を基準にチェックして問題点をリスト化して下さい。"
